01. Descrição do Problema

Por que precisamos de iluminação adaptativa?

💡

O Problema

Ambientes como laboratórios, escritórios e salas de estudo precisam manter níveis de iluminação estáveis e adequados independentemente das variações de luz natural (entrada de sol, nuvens, hora do dia). O controle manual de iluminação é ineficiente, gera desperdício de energia e não garante conforto visual contínuo.

🎯

A Solução

O projeto simula um sistema de iluminação inteligente para ambientes fechados, onde um sensor LDR monitora continuamente a luminosidade ambiente e um algoritmo de controle PI ajusta automaticamente o brilho de uma fita de LED 12V para manter um setpoint definido pelo usuário.

⚡

Importância: Eficiência energética, conforto visual, autonomia, conectividade via dashboard e MQTT, e escalabilidade para automação predial.

03. Tecnologias Utilizadas

Hardware & Software

🎛️

ESP32 DevKit

Microcontrolador dual-core 240 MHz com WiFi integrado. Utiliza ADC de 12 bits e PWM digital via ledcAttach.

☀️

LDR & DHT11

Sensores para leitura de luminosidade (LDR) via ADC e temperatura/umidade (DHT11).

⚡

MOSFET & LED 12V

MOSFET IRLZ44N atua como chave de potência para controlar via PWM a fita de LED 12V com segurança.

🌐

MQTT & WiFi

Comunicação IoT via HiveMQ Cloud com TLS. Payload JSON trocado a cada 1.5s.

🧮

Controle PI & EMA

Controle PI sintonizado por Ziegler-Nichols com filtro EMA (Exponential Moving Average) para suavizar ruído no LDR.

💻

Dashboard Web

Interface responsiva em HTML/CSS/JS usando Chart.js e MQTT.js via WebSocket.

C++ (Firmware Principal - Loop PID) 
// ── Controlador PI com Anti-Windup ──
double erro = setpointLux - luxReal;

// Proporcional
double P = kp * erro;  // kp = 3.15

// Integral com anti-windup
erroAcumulado += erro * dt;
erroAcumulado = constrain(erroAcumulado, -255.0 / ki, 255.0 / ki);
double I = ki * erroAcumulado;  // ki = 18.9

// Saída PWM (0–255)
pwmAtual = constrain((int)(P + I), 0, 255);
ledcWrite(LED_PWM_PIN, pwmAtual);
04. Desenvolvimento

Etapas do Projeto

Etapa 1

Definição do problema e requisitos

Análise das necessidades de controle de luminosidade e escolha do ESP32 pela disponibilidade de ADC, PWM, WiFi integrado.

Etapa 2

Projeto e montagem do hardware

Definição do circuito, divisor de tensão LDR, MOSFET IRLZ44N, montagem na protoboard e verificação.

Etapa 3

Firmware básico e Calibração

Leitura analógica (GPIO 34), PWM via ledcAttach e calibração da curva LDR → Lux via regressão.

Etapa 4

Controlador PI

Sintonização de Kp e Ki por Ziegler-Nichols. Implementação de anti-windup no integrador.

🔧

Dificuldades contornadas: Ruído no ADC resolvido com filtro EMA (α=0.1), Ground comum adicionado para o MOSFET funcionar, instabilidade do PI controlada com sintonia fina.

05. Resultados Obtidos

Análise e Métricas

O sistema demonstrou capacidade de manter a luminosidade estável no setpoint configurado, com tempo de acomodação de 3 a 5 segundos e erro de regime menor que 10 lux.

07. Conclusão

Reflexões e Evoluções

O projeto atingiu os objetivos?

Sim. O sistema LuxControl demonstrou com sucesso o controle autônomo de luminosidade utilizando um controlador PI sintonizado, com comunicação IoT via MQTT e visualização em tempo real por dashboard web. Os três modos de operação (automático, manual e desligado) funcionaram conforme especificado.

O que poderia ser melhorado?

  • Substituição do LDR por um sensor calibrado (ex: BH1750 I²C)
  • Substituição do DHT11 pelo DHT22
  • Armazenamento histórico em banco de dados (InfluxDB + Grafana)

Evoluções futuras

  • Automação predial com controle centralizado
  • Integração com Alexa/Google Assistant via MQTT
  • Monitoramento de consumo real (ex: INA219)
